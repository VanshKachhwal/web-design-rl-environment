# Project Brief — Web Design Replication RL Environment

> This is the verbatim task we are building against, lightly structured for reference.
> Source: project assignment. Captured 2026-05-30.

## Goal

Create a **scalable pipeline that generates RL environments**. These environments
test a coding agent's ability to **replicate a multi-page web design in code**
(ignore functionality — design only).

- Built on the **Harbor framework** (see the [grading runbook](./runbooks/harbor_grading.md)).
- Tasks are tested using **Claude Code with Opus 4.7**.

## Available infrastructure

- **Modal** access is available — we can run code/generation/grading on the cloud
  instead of pushing load onto the local machine.
- **Anthropic API key** is available (in `.env`) for generation and/or LLM-as-judge grading.

## Recipe design

Create a pipeline that can produce, **at scale**, tasks of this type:

> The agent is given **screenshots of a website**, uses them to **replicate the design
> in HTML + CSS**, and we have a **good way of grading** the model's performance.

We have a lot of freedom in how to set up this pipeline. Think creatively.

### Hard requirements

1. **Stable recipe** — the pipeline must reliably produce working tasks.
2. **Continuous grading** — grade performance on a **spectrum** for each generated task
   (continuous, *not* discrete pass/fail). Higher rewards must correspond to better designs.
3. **No crawling existing websites — not allowed.** Websites must be **generated from
   scratch**. (Do not worry about budget/cost if using a model provider to generate them.)
4. Each generated website must have **at least 5 pages**.
5. **Good distribution** of websites / types of websites.
6. **Functionality is out of scope** — judge design only.

### What matters most

> **There is nothing more important than showcasing research taste and good decisions
> when building these environments.** Get Part 1 done *really well* before moving on.
> A very high-taste trial with only Part 1 finished beats a rushed trial with all 3 parts.

Key questions to answer well:

1. **What is your grading?** The model learns the grading logic — bad grading just injects
   noise and makes the model worse. Ensure higher rewards = better designs under your logic.
2. **How complex, human-like, and varied** are the websites generated by your model?
3. **What behaviors** do you observe in the models / what **learnings** emerge?

## Part 2 — Animations

Beyond normal websites, add **animations**. Judge the model's ability to **perfectly
replicate** these animations. **Video recordings** may be passed to the model alongside
screenshots for this.

## Part 3 — Multiple Frameworks

HTML + CSS is one way to build sites. Extend the pipeline to also support:

1. **React JS + CSS**
2. **React JS + Tailwind CSS**
3. **Solid JS + Tailwind CSS**

## Deliverables

- **Recipe code** (automation pipeline) — shared via a **public GitHub link**.
  - Must include **documentation on how the problem was thought through** and how
    different aspects were solved (point to it in the GitHub README). Writing the docs
    with Cursor/Claude Code is fine — the goal is to **show how decisions were made**.
- **At least 10 final tasks** created using the recipe.
  - Chosen to **showcase the distribution and complexity** of the tasks. These 10 + the
    pipeline code are what gets judged.
  - Plus a **visual report of results**: running Claude Code with Opus 4.7 **10 times**
    on a task, how well does the grader score the results? Explain **why higher grades
    correspond to better design replications**.
  - Explain **common patterns the model struggles with** in these environments.
- **If Part 2 is done** — include some animation tasks in the environment.
- **If Part 3 is done** — give a **benchmark** of model performance across frameworks.
  Bonus: explain the patterns the model struggles with per framework type.

## General advice

1. **Finish previous parts fully and well** before continuing to the next.
2. **Over-communicate.** The more visible the reasoning, the better — surface thoughts
   and decisions throughout.
