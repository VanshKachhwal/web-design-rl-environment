Authentication

Required

CloudNest uses Bearer tokens for API authentication. Generate an API key from your team's dashboard under Settings → API Keys. All requests must include the token in the Authorization header.

# Include your API key in every request
curl -X GET "https://api.cloudnest.eu/v3/documents" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
⚠

Keep your keys secure

Never expose API keys in client-side code or public repositories. Use environment variables and rotate keys every 90 days for enterprise compliance.

Rate limits are applied per API key. Response headers include your current usage. Enterprise plans receive dedicated throughput pools.

Error Handling

The API returns standard HTTP status codes. Error responses include a machine-readable code and a human-friendly message.

// Example error response
{
  "error": {
    "code": "document_not_found",
    "message": "No document with ID 'abc-123' exists.",
    "status": 404
  }
}
Status Meaning Common Cause
400Bad RequestMalformed JSON or missing required field
401UnauthorizedMissing or invalid API key
403ForbiddenInsufficient permissions for this resource
404Not FoundResource does not exist
429Too Many RequestsRate limit exceeded
500Server ErrorInternal issue — contact support

Documents

Create, retrieve, update, and delete documents within your workspaces. Documents support Markdown content, metadata tags, and full version history.

GET /documents List all documents

Returns a paginated list of documents in the authenticated workspace. Supports filtering by tag, status, and date range.

?page=1 ?limit=25 ?tag=onboarding ?status=published
GET /documents/:id Retrieve a document

Fetch a single document by its unique identifier. Includes content body, metadata, collaborator list, and version history.

POST /documents Create a document

Create a new document with Markdown content, optional tags, and workspace assignment. Returns the created resource with a generated ID.

PUT /documents/:id Update a document

Replace the content and metadata of an existing document. A new version is automatically created in the document's history.

DELETE /documents/:id Delete a document

Soft-deletes a document. It can be restored within 30 days from the workspace trash. After 30 days, the document is permanently removed.

Request Body

POST /documents

{
  "title": "Onboarding Guide",
  "content": "# Welcome\n\nThis guide...",
  "workspace_id": "ws_8f3k2",
  "tags": ["onboarding", "hr"],
  "status": "draft"
}

Response 201 Created

{
  "id": "doc_q9x7m",
  "title": "Onboarding Guide",
  "version": 1,
  "created_at": "2024-11-15T09:32:00Z",
  "created_by": "usr_a1b2c",
  "status": "draft"
}

Workspaces

Workspaces are organizational containers for documents. Each workspace has its own set of collaborators, permissions, and branding settings.

GET /workspaces List workspaces for the authenticated team
POST /workspaces Create a new workspace
PUT /workspaces/:id Update workspace settings and branding
GET /workspaces/:id/members List workspace members and their roles

Users & Teams

Manage team members, roles, and permissions. Invite new collaborators, assign granular access levels, and audit activity logs.

GET /team/members List all team members
POST /team/invitations Invite a new member by email
PATCH /team/members/:id/role Update a member's role (admin, editor, viewer)

Webhooks

Subscribe to real-time events from your CloudNest workspace. Webhooks deliver JSON payloads to your endpoint within seconds of an event occurring.

⚡

document.created

Fires when a new document is created in any workspace.

✏

document.updated

Fires when document content or metadata is modified.

🗑

document.deleted

Fires when a document is moved to trash or permanently deleted.

👤

member.invited

Fires when a new team member invitation is sent.

📦

workspace.updated

Fires when workspace settings or branding are changed.

🔍

document.published

Fires when a draft document status changes to published.

// Webhook payload example
{
  "event": "document.published",
  "timestamp": "2024-11-15T14:22:08Z",
  "data": {
    "document_id": "doc_q9x7m",
    "workspace_id": "ws_8f3k2",
    "published_by": "usr_a1b2c",
    "version": 3
  }
}

Search

Full-text search across all documents in your team. Supports filters, highlights, and relevance scoring powered by CloudNest's European-hosted search infrastructure.

GET /search?q=onboarding setup&workspace=ws_8f3k2&limit=10

// Response
{
  "total": 23,
  "results": [
    {
      "id": "doc_q9x7m",
      "title": "Onboarding Guide",
      "score": 0.94,
      "highlight": "...complete <em>onboarding setup</em> for new..."
    }
  ]
}
Parameter Type Description
qstringSearch query (required, 1–256 characters)
workspacestringFilter results to a specific workspace ID
tagstringFilter by document tag
limitintegerResults per page (default: 20, max: 100)
offsetintegerPagination offset
