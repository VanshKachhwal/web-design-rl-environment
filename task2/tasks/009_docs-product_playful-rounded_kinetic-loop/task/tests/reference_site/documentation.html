Installation

Install the CloudNest CLI to manage your documentation spaces from the terminal. The CLI supports macOS, Linux, and Windows.

Step 1 — Install via npm

# Install the CloudNest CLI globally
npm install -g @cloudnest/cli

# Verify the installation
cloudnest --version
v3.8.2

Step 2 — Authenticate

Log in with your CloudNest account. This stores a secure token locally for future commands.

cloudnest login
# Opens your browser for SSO authentication
✓ Authenticated as anna@acme-corp.eu

Step 3 — Initialize a space

Create a new documentation space linked to your project repository.

cloudnest init my-project-docs
# Select a template:
❯ Blank Space

Configuration

CloudNest uses a simple JSON configuration file at the root of your project. Here are the most common options:

Property Type Default Description
space string — Unique identifier for your documentation space
source string ./docs Path to your documentation source files
theme string default Visual theme — choose default, minimal, or enterprise
locales array ["en"] Supported languages for multi-locale documentation
versioning boolean true Enable automatic version snapshots on each deploy
// cloudnest.config.json
{
  "space": "my-project-docs",
  "source": "./docs",
  "theme": "enterprise",
  "locales": ["en", "de", "fr"],
  "versioning": true,
  "search": { "enabled": true, "indexDepth": 3 }
}

Core Concepts

Understand the building blocks that make CloudNest powerful and flexible.

📂

Spaces & Pages

Spaces are top-level containers for related documentation. Each space holds pages organized in a customizable tree structure with drag-and-drop reordering, nested sub-pages, and automatic breadcrumb navigation.

Unlimited nesting Drag & drop
🔀

Versioning

Every deploy creates an immutable version snapshot. Readers can switch between versions via a dropdown, and you can diff any two versions side-by-side. Supports semantic versioning and branch-based previews.

Semantic versions Branch previews
🔒

Permissions & Roles

Define who can view, edit, or admin each space. CloudNest supports SAML SSO, SCIM provisioning, and custom role definitions. Set page-level visibility for internal-only or public documentation.

SSO / SAML SCIM
🔍

Search & Discovery

Full-text search across all spaces with typo tolerance, faceted filtering, and instant results. Search indexes update within seconds of publishing. Supports code-block search and multi-language stemming.

Instant results Multi-language
