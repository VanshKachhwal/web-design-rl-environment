# Task 2 — animation tasks (preliminary)

Ten curated **animation** tasks (`000`–`009`) generated by the Task-2 scaling pipeline
and evaluated with **Claude Code + Opus 4.7, 10× each**. This is the Part-2 counterpart
of the main [`tasks/`](../../tasks/) set — but **preliminary**: Part 2 was a time-boxed
attempt, so these are a working showcase, not the polished final deliverable (see
[`../README.md`](../README.md) for the honest scope, the grader, and the known
limitations — chiefly the fixed 6-frame filmstrip).

Each folder is a self-contained bundle:

- `task/` — the runnable Harbor task (the agent's reference **filmstrips** in
  `environment/reference/`, the baked grader in `tests/`, `task.toml`).
- `report.md` — the visual report of Claude Code + Opus 4.7 run **10×**: the five data
  sections **plus** the §6/§7 **animated-GIF galleries** (reference vs candidate motion).
  GIFs (`images/*.gif`) and plot PNGs render inline on GitHub.
- `scores.json` / `scores.csv` — the normalized per-trial score contract.
- `README.md` — per-task summary (narrative deferred).

Reward is `mean(static_design, motion, animation_judge)` per page, averaged over the 5
pages. **`motion` is the weakest term on every task** (≈0.20–0.56) — the candidates
reproduce the static design and animate, but rarely match the reference's exact motion
timing/placement; the strict `motion` term (and the 6-frame sampling) penalises that.
`static_design` (the reused Task-1 grader, ≈0.67–0.78) and `animation_judge` (the soft
VLM term, ≈0.50–0.57) sit higher. Full per-task breakdown in each `report.md`.

## Coverage

- **Archetypes (10):** agency-portfolio, docs-product, ecommerce-store, editorial-blog, event-conference, local-service, nonprofit-civic, personal-resume, restaurant-hospitality, saas-landing
- **Aesthetics (10):** brutalist, corporate-flat, dark-techy, glassmorphism, luxury-serif, neo-brutalist, playful-rounded, retro-y2k, swiss-editorial, warm-organic
- **Animation styles (5):** elegant-reveal, kinetic-loop, playful-bounce, smooth-fade, snappy-slide
- **Pages:** 5 per site (Task 2 fixes page count at 5 for now).

The seed walk gives **all 10 archetypes × all 10 aesthetics** (each once) crossed with the
5 animation styles (each twice) — the same stratified-coverage idea as Task 1, plus the
animation axis.

## How to use

**View a report** — open `report.md` on GitHub (renders inline), or locally:

```bash
open task2/tasks/000_saas-landing_swiss-editorial_smooth-fade/report.md
```

**Re-run a task** (Claude Code + Opus 4.7 ×10, via the Task-2 eval launcher):

```bash
PYTHONPATH=task2:src .venv/bin/python -m webdesign_rl_anim.evaluate_anim \
    --tasks task2/tasks/000_saas-landing_swiss-editorial_smooth-fade/task --executor modal --yes
```

See [`../README.md`](../README.md) for how the animation grader, the two-pass generator,
the lean gate, and the report pipeline were built.
