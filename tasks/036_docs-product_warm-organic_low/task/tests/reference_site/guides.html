Getting Started

Everything you need to go from zero to a running project. This guide walks through prerequisites, installation, and your first build.

Prerequisites

Before you begin, make sure your development environment meets the following requirements. You will need a modern operating system, a terminal application, and a text editor or IDE of your choice.

System Requirements

64-bit operating system (macOS 12+, Ubuntu 20.04+, or Windows 10+). At least 4 GB of RAM and 2 GB of free disk space.

Tooling

Node.js v18 or later, npm or yarn package manager, and Git for version control. A modern browser for testing.

Installation

Install the core package using your preferred package manager. The CLI tool will be available globally after installation.

Terminal 
# Install with npm
npm install -g @platform/cli

# Or install with yarn
yarn global add @platform/cli

# Verify the installation
platform --version
# Output: v3.2.1

Once installed, verify everything is working by running the version command. You should see the current version number printed to your terminal.

Configuration

Create a configuration file in the root of your project. This file controls build behavior, output paths, plugin loading, and environment-specific overrides.

JSON 
{
  "name": "my-project",
  "version": "1.0.0",
  "output": "./dist",
  "plugins": [
    "@platform/plugin-optimize",
    "@platform/plugin-analytics"
  ],
  "env": {
    "production": {
      "minify": true,
      "sourceMaps": false
    }
  }
}

For a complete list of all configuration options, see the API Reference page.

Build Your First Project

Follow these five steps to scaffold, develop, and preview a working project from scratch.

1

Initialize the Project

Run the init command to scaffold a new project directory with the recommended folder structure, starter templates, and a pre-configured settings file.

2

Define Your Data Model

Open the schema file and declare your entities, relationships, and validation rules. The CLI will auto-generate type definitions and migration files from your schema.

3

Add Your First Component

Create a component using the generate command. This produces a template, logic module, and test file following the conventions established by your configuration.

4

Start the Dev Server

Launch the local development server with hot-reload enabled. Changes to any source file will be reflected instantly in your browser without a full page refresh.

5

Build and Preview

Run the build command to produce an optimized production bundle. Use the preview command to serve the built output locally and verify everything works before deploying.
